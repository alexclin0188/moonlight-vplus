# 新版串流界面 — 实施计划总览

## 核心约束

- **旧代码零修改**：`com.limelight.Game`、`GameMenu`、`FloatBallHandler`、`FloatBallManager` 均不动
- **新代码独立**：全部放在 `com.alexclin.moonlink.stream.*` 包下，Compose 实现
- **复用底层引擎**：直接引用 `NvConnection`、`ControllerHandler`、解码器等 `com.limelight` 类
- **双线并存**：`com.alexclin.moonlink` 包下入口**直接跳转新 StreamActivity**；`com.limelight.PcView`（旧主页）**继续跳转旧 Game.kt**。无需 feature flag，两套页面独立运行，方便对比代码实现

## 关键决策（已确认）

1. **串流引擎**：直接引用 `NvConnection` / `ControllerHandler` 等 `com.limelight` 底层类（约 200-300 行初始化代码复制）
2. **详情页导航**：Compose `AnimatedContent` + `slideInHorizontally` 水平滑动切换
3. **按键映射编辑器**：先复用旧 `ElementController.editMode()` 原生编辑器

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| StreamEngine 初始化遗漏关键步骤导致串流失败 | 阶段0先做最小可串流验证（视频+音频通了再往下走） |
| Compose Window层与 SurfaceView 触摸事件冲突 | 面板区域拦截触摸事件，面板外侧穿透到 SurfaceView |
| 面板频繁动画导致串流帧率下降 | 动画使用硬件加速，阶段6做性能profiling |
| 旧Game.kt的ControllerManager被多Activity同时持有 | StreamEngine独立创建ControllerManager实例 |
| 自定义按键JSON格式与新/旧页面不兼容 | CustomKeyRepository完全复用旧SharedPreferences key和JSON结构 |
