# 基线副本 - MoonLink 包迁移与清理方案

## 记录时间
2026-06-21

## 原始需求
将 MoonLink 应用从旧的包结构迁移到新的独立包名 `com.alexclin.moonlink.android`，同时清理所有向后兼容代码、简化数据库初始化、精简导入导出功能。

---

## 本轮对话已完成的调研工作

### 1. 按键映射 Bug 修复（已完成 4 轮）
- 第1轮：修复 3 个 bug（SubPanelContainer derivedStateOf、reloadOverlay 空保护、removeCallbacksAndMessages 范围过大）
- 第2轮：修复 5 个 bug（摇杆死区元素级灵敏度、摇杆双击 middleValue 前缀处理、手柄重复触发延时、滚轮 SU/SD 重复、轮盘边界 inclusive/exclusive）
- 第3轮：修复 6 个 bug（默认元素 D-Pad 非法"d"前缀、parseValueToFlag 进制混用、D-Pad g-前缀方向值忽略、轮盘 gb{id} 空存根、内置摇杆 L3/R3 middleValue 缺失、k0x 死代码）
- 第4轮：确认修复全部正确，额外发现 sendStickDirection 仍走 sendKeyboardKey（不影响实际场景）

### 2. 代码结构调研
- `com.alexclin.moonlink`: ~68 个源文件（按键映射 UI/引擎/流页面）
- `com.limelight`: ~150 个源文件（Moonlight 流核心/输入/发现/UI）
- 包名变更将影响所有 ~218 个源文件 + AndroidManifest + Gradle 配置

### 3. 数据库调研
- `SuperConfigDatabaseHelper.java`: `DATABASE_VERSION = 11`（重置为 **1**），6 个旧版本常量，完整 `onUpgrade()` 链从 v1→v11
- `onCreate()` 已包含当前 v11 schema（移入新版本时 schema 不变，只改 VERSION=1）
- `upgradeExportedConfig()` 用于导入旧 Crown 配置时的迁移

### 4. 兼容代码调研
- `ControllerManager.kt`：整个类（旧 Crown 桥接）
- `StreamEngine.kt`：`controllerManager` 属性 + `initializeControllerManager()`
- `SchemeUtils.kt`：`syncToControllers()` 方法
- 旧版本选单相关代码、列名向下兼容 fallback

### 5. 导入导出调研
- `SchemeImportExport.kt`：JSON 格式导入导出，支持新旧方案管理
- 新格式 `.mkmp`：按键映射方案导入/导出
- 旧格式 `.mdat`：完整旧版本数据导出（可移除）
