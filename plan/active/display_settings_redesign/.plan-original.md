# 显示设置子面板重构计划

> 创建日期：2026-06-16 | 状态：ready | 版本：1.2

---

## 一、需求概述

将现有 `DisplaySection` 完全重写为**两分组布局**。

### 分组1：帧率与画面
- 帧率选择（折叠RadioButton + 解锁FPS + 自动帧率参考旧代码`addNativeFrameRateEntry`）
- 码率选择行（[自动][2M][8M][20M][自定义]：自动→智能码率参考旧`AdaptiveBitrateService`；自定义→SeekBar参考旧`BitrateCardController.adjustBitrate`）
- 流量估算小字（kbps/8/60）
- HDR + HDR高亮度（保留旧逻辑）
- VDD虚拟显示器 + 外接显示器（从旧显示器分组移入）
- 输出缓冲区（SeekBar 1~5帧）
- MTK专属选项
- 拉伸/反转/旋转 3画面开关

### 分组2：显示器
- 显示器列表（`NvHTTP.getDisplays()` → RadioButton切换）
- 分辨率设置（参考旧代码：6预置分辨率+Native+自定义分辨率`custom_resolutions` SP；首选0x5506通道运行时修改，降级`changeResolution→recreate`）
- DPI缩放（运行时POST调用，不持久化）

---

## 二、关键设计决策

| 问题 | 决策 |
|------|------|
| HDR设置 | 保留在帧率与画面分组 |
| "自动"帧率含义 | 参考旧代码`StreamSettings.addNativeFrameRateEntry`，检测设备原生刷新率 |
| 码率"自动" | 参考旧`AdaptiveBitrateService`：prefConfig.enableAdaptiveBitrate + abrMode + notifyManualOverride模式 |
| 分辨率设置 | 参考旧代码：预置+Native+自定义分辨率(旧`custom_resolutions` SP)；首选0x5506降级recreate |
| VDD/外接显示器 | 移入帧率与画面分组 |
| DPI缩放持久化 | 不持久化，仅运行时调用 |
