# 按键映射方案执行计划 — Baseline

> 创建日期：2026-06-18 | 状态：prepare | 版本：1.1
> 参考设计文档：`MoonLink_Detailed_Design.md` 第5章（串流界面）和第6.7节（按键配置管理）

---

## 一、设计目标

将旧 Crown（王冠）按键映射体系优雅地迁移到 MoonLink 新 UI 框架中：

1. **命名重塑**：王冠 → 按键映射方案 / 操控方案，Preference key 独立新 key，**不做向后兼容**
2. **UI 重写**：方案选择页面和编辑器用 Compose 全屏实现（编辑器核心复用旧 ElementController）
3. **虚拟手柄合并**：旧 VirtualController 配置并入按键映射方案体系
4. **配置管理**：Settings Tab 新增 Compose 页面，替代旧 `category_crown_features`
5. **文件格式**：新增 `.mkmp` 简化导出格式，`.mdat` 保留

## 二、与 streamplan 的关系

- streamplan 是串流界面大重构（StreamActivity + Compose 面板体系）
- 本计划是 **KeyMapping 专项深化**，作为**独立并行工作流**
- 前置依赖：streamplan Phase 2（SubPanelContainer + MainListView 存在）——已就绪
- KeyMappingSection（开关 + 3 子入口）在 streamplan Phase 4.3 已实现空壳，本计划需连接真实逻辑

## 三、技术约束

1. **旧代码零修改**：`com.limelight.Game`、`GameMenu`、`FloatBallHandler` 均不动
2. **Crown 核心逻辑复用**：`ControllerManager`、`ElementController`、`SuperConfigDatabaseHelper` 保持原封装不变
3. **新代码在 `com.alexclin.moonlink.stream` 包下**，与现有 streamplan 代码同包
4. **旧入口仅做参考**：旧 `com.limelight.PcView` → 旧 `Game.kt` 仍可用，但后续将被废弃，不做兼容性维护
